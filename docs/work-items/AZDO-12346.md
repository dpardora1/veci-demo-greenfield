## Descripción funcional (tal como llega del cliente)

> Como cliente de VECI que está terminando una reserva, quiero poder introducir un **código promocional** durante el checkout para que se me aplique el descuento correspondiente al precio total antes de confirmar la reserva.

Casos comentados en el workshop con negocio:

- Códigos de un único uso por usuario y por reserva.
- Códigos con vigencia (rango de fechas) y límite de canjeos totales.
- Códigos restringidos por destino y/o tipo de excursión.
- Apilado con otras promociones: depende del tipo del código.
- Descuento porcentual o fijo, con tope opcional.

## Restricciones conocidas

- Debe respetar el flujo de checkout actual definido en `SPEC-2026-0042` (reserva de excursión opcional).
- Auditoría legal: cualquier uso de código debe quedar trazado para reclamaciones.
- Tiempo de respuesta: la validación del código debe completarse en <300 ms p95 para no degradar la experiencia.

## Criterios de aceptación de alto nivel

- Si introduzco un código válido, el precio se recalcula y se muestra el ahorro.
- Si el código es inválido / caducado / agotado, se muestra error específico y el precio no cambia.
- Si quito el código, el precio vuelve al original.
